Monitoring Cheat Sheet · A4

Deploy, Collect, Alert, Update auf einer Seite

v1.5.22 · Druckfreundlich

Verdichtete Referenz für Betrieb und Störungsanalyse. Diese Seite ist für A4-Druck optimiert und zeigt nur die Kernpfade, Skripte und Parameter.

Windows Basis

Pfad: C:\ProgramData\monitoring-agent
Tasks: collect, update

Linux Basis

Pfad: /opt/monitoring-agent
Cron: collect, update

Server Ziel

Ingest: /api/v1/agent-report
Updates: /updates

End-to-End Ablauf

01

Deploy

Install-Skript legt Config, Pfade, Queue und Scheduler/Cron an.

02

Collect

collect_and_send sammelt Host-, FS-, DB- und SAP-Daten.

03

Queue

Bei Verbindungsfehlern wird lokal zwischengespeichert.

04

Ingest

Server validiert JSON, Pflichtfelder und Auth/Grace-Mode.

05

Alert

DB-Write, Thresholds, Routing, Mail/Telegram und Reminder.

06

Update

self_update lädt Skripte und Agent-Version neu vom Server.

Wichtige Skripte

Windows

  • install_agent.ps1 für Neuinstallation
  • bootstrap_agent.ps1 für Repair/Rebootstrap
  • collect_and_send.ps1 für Datensammlung
  • self_update.ps1 für In-Place-Update

Linux

  • install_agent.sh für Neuinstallation
  • collect_and_send.sh für Datensammlung
  • self_update.sh für Update
  • repair_agent_conf.sh für Config-Fix

Wichtige Parameter

Pflicht

ServerUrl / --server-url bestimmt das Zielsystem.

Auth

ApiKey / --api-key für abgesicherte Agent-Calls.

Rhythmus

IntervalMinutes / --interval-minutes für Collect-Takt.

Updates

UpdateHours / --update-hours für Self-Update-Takt.

Sonderfall

RawBaseUrl, --collect-script-url, --insecure-tls nur für Bootstrap-/TLS-Sonderwege.

Schnellbefehle

Windows Collect:
C:\ProgramData\monitoring-agent\collect_and_send.ps1 -NoJitter

Windows Update:
powershell.exe -ExecutionPolicy Bypass -File C:\ProgramData\monitoring-agent\self_update.ps1

Linux Collect:
sudo /opt/monitoring-agent/collect_and_send.sh --no-jitter

Linux Update:
sudo /opt/monitoring-agent/self_update.sh

Debug-Reihenfolge

  • Deploy-Fehler: Installation, Config, Task/Cron prüfen
  • Keine Daten: Collector manuell starten, Logs und Queue prüfen
  • Reject im Server: API-Key, Pflichtfelder, Grace-Mode prüfen
  • Kein Alarm: Threshold, Blacklist, Mute/Ack, Routing prüfen
  • Kein Update: /updates, lokale Version und Update-Logs prüfen
